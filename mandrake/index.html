RT Activity Prediction for Prime Editing

Predicting which reverse transcriptases work for prime editing using protein language models (ESM-2) and structural features extracted from AlphaFold structures. Built at Mandrake Bioworks.

How it works

Evaluation. Leave-One-Family-Out (LOFO) cross-validation holds out all members of one evolutionary RT family per fold, rotating across all 7 families. Standard random k-fold would mix family members across train and test, letting the model learn "if it looks like the Retroviral clade, it's probably active" — inflating apparent performance while testing nothing about generalisation. LOFO forces the model to predict activity for a family it has never seen, which is exactly what happens when a company screens an RT from a novel organism.

Architecture. The pipeline uses a two-stage design: a Logistic Regression classifier produces P(active) probabilities that feed into PR-AUC, and an XGBoost + CatBoost regressor ensemble predicts PE efficiency values that feed into Weighted Spearman. Both outputs are converted to fractional ranks [0,1] before blending at a 0.66/0.34 ratio. Rank blending rather than score multiplication avoids the calibration problem — LR probabilities and regression predictions live on incompatible numerical scales, so direct averaging is meaningless. The blend weight was tuned by sweeping 0.00–1.00 in 0.01 steps under LOFO CV.

Key features. FoldSeek TM-scores (structural similarity to known RTs) go into both models — they are the primary activity signal. Palm domain geometry features (thumb_fident: ρ=0.901 with PE efficiency; n_hairpins_found: ρ=0.816) go into the regressor only: they capture processivity-relevant local structure that is absent from global TM-scores and would add noise to the classifier. Thermal stability features (t55, t60, t65) also go into the regressor, reflecting the correlation between thermostability and enzymatic activity. ESM-2 PCA components (5 dims, fitted inside each LOFO fold) go into both models, contributing evolutionary context without leaking test-set information into the embedding space.

Experiment discipline. v10–v14 represents 20+ controlled experiments. v12 swept LR regularisation strength and PCA dimension count. v13 tested 13 candidate features one at a time in OOF — features were only added if they improved the held-out CLS on all 7 folds. v14 combined the winning features from v13. A key lesson from v11: features that help classification (e.g. hbonds_per_res) can hurt WSpearman when added to the regressor, because they introduce signal that is discriminative for active/inactive but uncorrelated with efficiency rank order. The regressor was kept pure — only features correlating with efficiency were allowed in.
