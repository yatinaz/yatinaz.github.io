💡 Aha! The "plug-and-play" architecture

Want to add Parkinson's? Create a new explainer, train a new model, and add one line to the task registry. The API server doesn't change. This is called open/closed principle.

# Lazy-load any task's components
def get_task_components(task_name: str):
    if task_name in _task_components:
        return _task_components[task_name]

    config = get_task_config(task_name)
    module = importlib.import_module(
        config['preprocessor_module']
    )
    Class = getattr(module,
        config['preprocessor_class']
    )
    components['preprocessor'] = Class()
    _task_components[task_name] = components
    return components
Plain English
1
Check the cache first — if we've loaded this task before, return instantly. No re-loading models every request.
2
Look up the config — which preprocessor, model, explainer does "schizophrenia_fmri" need?
3
Dynamically import — instead of hardcoding imports, load the right class by name. Lets us add new tasks without touching this function.
4
Instantiate and cache — create the preprocessor object and save it for next time.

Module 3

Reading the Brain

How raw MRI data — essentially millions of magnetic measurements — becomes 15 meaningful brain connectivity numbers.

💡 The Vinyl Record Metaphor

Imagine the brain is a city with 116 neighborhoods. Each neighborhood has a radio station (brain region) broadcasting a signal over time. We measure how in sync two stations are — do they play the same songs at the same time? That synchrony is functional connectivity. In schizophrenia, certain stations stop syncing.

📐
Step 1: Resample
The scan is resampled to 4mm isotropic — a standard grid size that matches the atlas. Like resizing a photo to standard dimensions before cropping.
🗺️
Step 2: Atlas Masking
Apply 116 brain region masks from the AAL atlas. Average all voxels inside each mask → 116 timeseries signals, one per brain region.
🎛️
Step 3: Filter Noise
Bandpass filter: keep only 0.01–0.1 Hz. Removes slow scanner drift (below 0.01 Hz) and fast heartbeat/breathing artifacts (above 0.1 Hz).
📊
Step 4: Correlate
Compute Pearson correlation for every pair: 116×115÷2 = 6,670 unique pairs. This is the functional connectivity matrix.
def compute_fc_matrix(roi_timeseries):
    n_rois = roi_timeseries.shape[1]  # 116
    fc = np.zeros((n_rois, n_rois))

    for i in range(n_rois):
      for j in range(i, n_rois):
        if i == j:
          fc[i, j] = 1.0
        else:
          corr, _ = pearsonr(
            roi_timeseries[:, i],
            roi_timeseries[:, j])
          fc[i, j] = corr
          fc[j, i] = corr

    triu = np.triu_indices(n_rois, k=1)
    return fc[triu]  # shape: (6670,)
Plain English
1
For each pair of brain regions — 116×116 is the full grid, but we only need the upper triangle (pairs, not duplicates).
2
A region perfectly correlates with itself — diagonal is always 1.0. We skip it.
3
Pearson correlation — measures synchrony between two region timeseries. Output: -1 (opposite) to +1 (perfectly in sync).
4
Mirror to lower triangle — correlation(A,B) = correlation(B,A), so fill both sides.
5
Extract upper triangle only — flattens the 116×116 matrix to 6,670 unique values. This is the feature vector.
🚨 The Preprocessing Rejection Guard

The code checks if uploaded data is already preprocessed (2mm voxels → MNI space, mean-centered, z-scored) and rejects it. Why? Because the model was trained on raw data that went through this exact pipeline. Double-preprocessing would corrupt all the correlation values.

Beyond the technical rejection: the model learned patterns from FC computed using this specific pipeline (4mm resampling, 0.01–0.1 Hz filter, AAL masker). Different preprocessing = different FC values = model comparisons are invalid.

🚨 The Bug That Caused v2.1 to Fail

Switching from Logistic Regression to SVM decreased AUC from 0.571 to 0.471. Why? With only 71 subjects and 2,110 features, SVM's RBF kernel overfit badly. The lesson: more powerful model ≠ better results when data is scarce.

💡 The Key Insight: Literature Beats Data

Going from 2,110 LASSO-selected features to 15 literature-curated pairs improved AUC from 0.571 → 0.614. With only 71 subjects, the model couldn't reliably learn which of 2,110 features mattered. Neuroscience literature already knew which 15 circuits to look at.

Module 5

The Trust Problem

Two bugs that were caught before deployment — and the statistical technique that stops scanners from cheating the model.

Bug 1: The Paradigm Mismatch

🚨 What happened

Version v5-mixed pooled healthy controls from a task-fMRI dataset (people performing a speech task) with schizophrenia patients from resting-state scans. AUC dropped from 0.614 → 0.524. The model learned to distinguish "brain during speech" from "brain at rest" — not healthy from schizophrenic.

🏃
Task fMRI
Brain doing something — speech, memory, movement. FC patterns reflect the task, not the person's baseline state.
😌
Resting-state fMRI
Brain at rest — no task. FC patterns reflect intrinsic brain organization. Required for psychiatric biomarkers.

Bug 2: Site Confound (Data Leakage)

🚨 What happened

Parkinson's controls came from one hospital (ds004302), patients from a different hospital (ds004392). Different MRI scanners have different "fingerprints" — the model learned to detect scanner make, not disease. This is called a confound or site effect.

The Fix: ComBat Harmonization

ComBat removes scanner-specific effects. Think of it like color-correcting photos taken on different cameras so they look like they came from the same source.

# ComBat: fit on train, apply to both
for fold, (tr, val) in enumerate(cv.split(X, stratum)):
    X_tr, X_val = X[tr], X[val]
    s_tr, s_val = sites[tr], sites[val]

    # CRITICAL: fit on training only
    gamma, delta = fit_combat(X_tr, s_tr)

    X_tr  = apply_combat(X_tr, s_tr,
                         gamma, delta)
    X_val = apply_combat(X_val, s_val,
                         gamma, delta)

    model.fit(X_tr, y[tr])
    auc = roc_auc_score(y[val],
        model.predict_proba(X_val)[:,1])
Plain English
1
5-fold cross-validation — split data into 5 chunks; train on 4, test on 1, rotate 5 times for an unbiased average AUC.
2
CRITICAL: fit ComBat only on training data — if you fit it on all data including the test fold, the test set influences how you transform training data = data leakage.
3
Apply to validation using train-derived params — remove scanner effects using only the statistics learned from training. No peeking at test data.
4
Train and evaluate — now the model sees scanner-removed features. AUC measures real disease detection, not scanner ID.
💡 The Compound Stratum Trick

Standard StratifiedKFold only balances disease labels across folds. With 2 sites, you also need site balance. The fix: create a compound label stratum = label × n_sites + site_index. Now each fold has subjects from both sites AND both classes.

Scenario
You fit ComBat normalization parameters on the entire dataset (all 326 subjects) before splitting into train/val folds. A colleague says this is fine because ComBat is "just normalization." Are they right?
Yes — normalization doesn't affect model training, only scaling
No — fitting on all data lets the validation subjects influence how training data is transformed, inflating your AUC estimate
It depends on the number of sites
Only a problem if sites are imbalanced
✓ Correct — this is "preprocessing leakage," one of the most common subtle bugs in ML pipelines. When ComBat fits on all data, the validation set's site statistics leak into the training transformation. The resulting AUC overestimates real-world performance because the model implicitly saw test data distribution during training.
Module 6

Why Should You Trust It?

A prediction without explanation is useless in medicine. Two complementary strategies for earning clinical trust.

📖
Literature Patterns
Rules encoded from published neuroscience: "frontotemporal connectivity should be high — if it's low (z-score > 2), flag it." Interpretable by any neurologist.
🔍
SHAP / Gradient Importance
What did this model actually use to make this decision? SHAP assigns credit to each brain connection. Independent of the literature.
💡 The Detective Metaphor

Literature patterns = the checklist a detective uses based on known crime patterns. SHAP = the actual fingerprints at this specific crime scene. When they match, confidence goes up. When they diverge, you might have found a new pattern nobody knew about.

Plain English
1
FCPattern is a rule card — one known brain circuit disruption from published neuroscience. Encoded as structured data, not free text.
2
roi_indices — the actual numbers in the 116-ROI AAL atlas, so the code knows exactly which correlation to look up.
3

4
direction_match — not just anomalous, but anomalous in the right direction. A connection that's unusually strong when literature says it should be weak → not a match.

💡 The Airport Metaphor

Nginx is the airport terminal. It doesn't process anything — it just reads the destination (/api/* → backend gate, everything else → frontend gate) and routes traffic. The backend and frontend never talk to each other directly.

The Async Job Pattern

fMRI preprocessing takes 30–120 seconds. If the API blocked waiting, your browser would time out. The solution: return a job ticket immediately, process in the background, and let the client poll for results.

@app.post("/api/upload")
async def upload_fmri(
    background_tasks: BackgroundTasks,
    file: UploadFile,
    task: str = "schizophrenia_fmri"
):
    job_id = str(uuid.uuid4())
    # Save file, return job_id immediately
    background_tasks.add_task(
        process_fmri, job_id, path, task
    )
    return {"job_id": job_id,
            "status": "queued"}

# Client polls every 2 seconds:
@app.get("/api/status/{job_id}")
async def get_status(job_id: str):
    return job_status[job_id]
    # {status, progress: 0.0→1.0, message}
Plain English
1
Generate a unique job ID — like a baggage claim ticket. The client uses this to ask "is my bag ready yet?"
2
Add to background queue — kick off the heavy processing without blocking. The function returns in milliseconds.
3
Return immediately — the response says "queued" with a job ID. The browser shows a loading spinner and starts polling.
4
Status endpoint — the client polls this every 2 seconds. Returns progress (0.0 to 1.0) and a human-readable message like "Extracting ROI timeseries..."
